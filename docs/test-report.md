# Краткий отчёт о тестировании API (SWAPI)

## 1. Что проверялось
- Получение данных персонажей (Luke Skywalker, Darth Vader) по ID и проверка ключевых атрибутов.
- Негативный сценарий: запрос несуществующего персонажа и корректность обработки ошибки.
- Планеты, на которых “побывал” персонаж, через цепочку `people → films → planets`.
  - Так как SWAPI не содержит прямой связи посещения персонажем конкретной планеты, проверка выполнена как “планеты, связанные с персонажем через фильмы”.
- Контрактные проверки нескольких эндпоинтов: `films`, `vehicles`, `starships`.

## 2. Ошибки и несоответствия
Критических дефектов не обнаружено.

Замечания/риски:
- В ответах встречаются значения `unknown` для числовых атрибутов, поэтому проверки типов должны учитывать этот кейс и клиент должен корректно его обрабатывать.
- «Планеты, где был персонаж» вычисляются через `films`, что увеличивает число запросов и время прогона.

## 3. Общее качество API
- API удобное для обучения и базового тестирования: простые GET-запросы, логичные связи между сущностями.
- Данные в целом структурированы; основной нюанс — особенности домена (`unknown`/строковые числа) и отсутствие прямой связи visited planets.
